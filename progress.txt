## Codebase Patterns (READ THIS FIRST ON EACH ITERATION)

### Architecture Overview
- Python 3.12+ with asyncio for all I/O
- Pydantic Settings for configuration
- aiohttp/aiosqlite for async operations
- PyTorch for LSTM, sklearn/xgboost/lightgbm for other models
- Structured JSON logging throughout

### Directory Structure
```
src/
├── core/config.py          # Pydantic Settings - all parameters
├── data/                   # Data layer (WebSocket, REST, DB, Gateway)
├── features/               # Feature calculation (86 canonical features)
├── ml/                     # ML models + ensemble + trainer
├── strategy/               # Regime detection, exits
├── risk/                   # Position sizing, exposure, risk validation
├── execution/              # Signal processing, orders, position sync
└── utils/                  # Fee calculator, drift detector
```

### Coding Conventions
- All async functions prefixed with `async def`
- All exchange operations are async
- Use `logging.getLogger(__name__)` for module loggers
- Type hints on all function signatures
- Dataclasses for structured data (Position, Candle, Order, etc.)
- `@dataclass(frozen=True)` for immutable data structures

### Gotchas & Learnings
1. **LightGBM degenerate trees:** Always validate num_leaves > 1 before using
2. **Position sync:** Exchange is always source of truth
3. **Close orders:** NEVER block close orders in risk manager
4. **Balanced training:** Critical for preventing SHORT bias
